# 在线刷题系统

一个纯前端开发的在线刷题网站，可直接部署在 GitHub Pages 上。支持练习模式与考试模式，错题自动收录，数据可导出导入。

## 项目由来

本项目由 AI 根据以下提示词自动生成：

> 帮我写一个在线刷题网站，有以下要求：
> 1. 纯前端技术开发，可以直接部署在 GitHub pages上，适配手机的竖屏和电脑的横屏
> 2. 支持单选，多选，判断，填空，简答题目。填空和简答在用户提交后显示比对界面，让用户自行判断所填内容是否正确，填空题由于空比较多，按大题号一起显示一整道题然后比对，在比对的时候可以选择自己哪个空错了
> 3. 分为练习模式和考试模式。练习模式可以手动选择是否打乱题目，每写完一道题自动展示答案，填空题和简答题都是每写完一道题进行比对。考试模式时将所有的题打乱，全部都答完再显示答案和按题进行比对。打乱题目时要把同种类型的题放在一起（不要出现一道单选，一道多选，然后又是单选的情况）
> 4. 错误的题自动保存在错题本中，方便随时复习
> 5. 进入网站后操作顺序示例：首页 -> 高中题库 -> 高一 -> 高一数学 -> 开始答题；也就是说拥有多套题组，题组里有多个题库，方便归类，同时错题本也可以按照题组和题库进行筛选
> 6. 在项目文件层面，题组和题库是按照文件夹层级关系来处理，当然可以拿一个配置文件去配置层级关系，但是一定一定要按照文件夹的层级关系处理，便于人类阅读
> 7. 同时写两个文档，一个给人类阅读，介绍每个文件夹都是干啥的，如何写题目，如何配置配置文件。一个给AI阅读，当我把这个文档和题目的word的文件一起发给ai的时候，ai可以自动帮我把word格式的题目转换成所需的配置文件的格式。要让ai理解题和答案的对应关系，同时只允许复制，文档里面没有的东西就设为空值，不允许自行编造或者解释
> 8. 添加示例题目，难度就小学一年级和二年级即可
> 9. 不需要任何表情，也就是emoji
> 10. 写出完整的readme，便于使用此网站，并注明我给你的所有提示词和这个项目是ai写的
> 11. 每道题在配置文件里都有解析的位置，但是可以留空，如果留空渲染成无即可
> 12. 因为信息都存储在用户浏览器缓存里，所以可以导出自己的存档，方便换设备使用
> 13. 性能一定要好，少bug，有精美的界面

## 功能特性

- **纯前端实现**：无需后端服务器，直接部署到 GitHub Pages
- **响应式设计**：完美适配手机竖屏和电脑横屏
- **五种题型**：单选题、多选题、判断题、填空题、简答题
- **练习模式**：逐题作答，即时反馈，可选是否打乱题目
- **考试模式**：全部答完后统一查看结果，打乱时同类型题目集中在一起
- **错题本**：自动收录错题，支持按题组和题库筛选，支持逐题复习
- **存档管理**：支持导出/导入 JSON 存档，方便换设备使用
- **文件夹层级**：题目数据按文件夹层级组织，便于人类阅读和手动管理

## 快速开始

### 1. 部署到 GitHub Pages

1. Fork 或下载本项目
2. 将项目文件推送到 GitHub 仓库
3. 在仓库设置中启用 GitHub Pages（选择 main 分支的根目录）
4. 访问生成的 Pages 链接即可使用

### 2. 本地预览

直接在浏览器中打开 `index.html` 文件即可。由于使用 `fetch` 加载本地 JSON，建议使用本地服务器：

```bash
# 使用 Python
python -m http.server 8000

# 使用 Node.js
npx serve .
```

然后访问 `http://localhost:8000`

## 使用指南

### 添加题目

1. 在 `data/` 目录下按照层级创建文件夹
2. 在每个层级放置 `config.json` 配置文件
3. 在题目集文件夹中创建 `questions.json`

详细说明请查看 `docs/human-docs.md`。

### 使用 AI 转换题目

如果你有 Word 格式的题目，可以将 `docs/ai-docs.md` 和 Word 文件一起发给 AI，AI 会自动转换为 `questions.json` 格式。

详细说明请查看 `docs/ai-docs.md`。

- Chrome / Edge / Firefox / Safari 最新版本
- iOS Safari / Android Chrome

## 数据存储说明

所有用户数据（包括答题记录和错题本）均存储在浏览器 `localStorage` 中。清除浏览器数据会导致数据丢失，请定期使用"存档"功能导出备份。
