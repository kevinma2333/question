# AI 题目转换文档（AI 阅读版）

## 任务说明

当你收到此文档和一个包含题目的 Word/文本文件时，你的任务是将 Word 中的题目转换为项目所需的 `questions.json` 格式。

## 核心规则（必须遵守）

1. **只允许复制**：文档中明确出现的内容，直接复制到 JSON 中。文档中没有的内容，对应的字段值必须设为空字符串 `""` 或空数组 `[]`。
2. **不允许编造**：不要自行添加解析、分值、选项编号。不要解释题目含义。不要修改题目原文。
3. **不允许自行解释**：如果题目有歧义，按原文原样录入，不做任何解释或修改。
4. **题型识别**：根据题目特征判断题型，见下方"题型识别规则"。
5. **空值处理**：如果某字段在原文中没有对应内容，一律设为空值。

## 题型识别规则

### 单选题（type: "single"）
- 特征：有 A/B/C/D 等选项，且只能选一个
- answer 格式：单个字母字符串，如 `"B"`
- options 格式：字符串数组，如 `["A. 选项1", "B. 选项2"]`

### 多选题（type: "multiple"）
- 特征：有 A/B/C/D 等选项，且可以选多个，题目中明确说明"多选"
- answer 格式：字母数组，如 `["A", "C"]`
- options 格式：同单选题

### 判断题（type: "judgment"）
- 特征：题目要求判断"正确"或"错误"，或"对"或"错"
- answer 格式：`true` 或 `false`
- options 固定为 `["正确", "错误"]`
- 如果原文答案是"正确"/"对"/"是"，设为 `true`；"错误"/"错"/"否"，设为 `false`

### 填空题（type: "fill"）
- 特征：题目中有空白处需要填写，通常用下划线、括号或"____"表示
- 处理方式：将题目中的空白处统一替换为 `____`
- answer 格式：字符串数组，每个元素对应一个空，按题目中空白出现的顺序排列
- 如果原文有 3 个空，answer 必须是长度为 3 的数组

### 简答题（type: "essay"）
- 特征：需要文字回答，没有固定选项
- answer 格式：字符串，参考答案
- 如果原文没有提供参考答案，answer 设为 `""`

## 字段映射规则

| JSON 字段 | 来源 | 空值处理 |
|-----------|------|----------|
| `id` | 自动生成，格式为 `q` + 序号，如 `q1`, `q2` | 不可为空 |
| `type` | 根据题型识别规则判断 | 不可为空 |
| `question` | 题目原文（不含选项和答案） | 不可为空 |
| `options` | 选项列表（单选/多选/判断） | 填空/简答题设 `[]` |
| `answer` | 原文给出的答案 | 没有则设 `""` 或 `[]` 或 `false` |
| `analysis` | 原文给出的解析/说明 | 没有则设 `""` |
| `points` | 原文给出的分值 | 没有则设 `5` |

## 题目与答案的对应关系识别

### 单选题对应关系
- 原文中通常有"答案：B"或"【答案】B"等标记
- 提取字母作为 answer
- 如果答案在选项中用括号标注，如"(B) 选项2"，提取 B

### 多选题对应关系
- 原文中通常有"答案：AB"或"【答案】A、C"等标记
- 提取所有字母组成数组，如 `["A", "B"]` 或 `["A", "C"]`
- 注意：多选题答案中字母之间可能有顿号、逗号或没有分隔符，统一提取为字母数组

### 判断题对应关系
- 原文中通常有"答案：正确"或"【答案】错"等标记
- 映射为 `true` 或 `false`

### 填空题对应关系
- 原文中通常有"答案：3"或每个空单独标注答案
- 按题目中空白出现的顺序，依次提取答案到数组
- 如果原文只给了一个总答案但题目有多个空，按顺序拆分

### 简答题对应关系
- 原文中通常有"参考答案：..."或"答案：..."
- 直接复制参考答案内容到 answer 字段
- 如果没有参考答案，answer 设为空字符串

## 输出格式示例

### 单选题转换示例

原文：
```
1. 1 + 1 = ?
A. 1  B. 2  C. 3  D. 4
答案：B
```

转换结果：
```json
{
  "id": "q1",
  "type": "single",
  "question": "1 + 1 = ?",
  "options": ["A. 1", "B. 2", "C. 3", "D. 4"],
  "answer": "B",
  "analysis": "",
  "points": 5
}
```

### 多选题转换示例

原文：
```
2. 下面哪些数字比3大？
A. 1  B. 4  C. 5  D. 2
答案：B、C
```

转换结果：
```json
{
  "id": "q2",
  "type": "multiple",
  "question": "下面哪些数字比3大？",
  "options": ["A. 1", "B. 4", "C. 5", "D. 2"],
  "answer": ["B", "C"],
  "analysis": "",
  "points": 5
}
```

### 判断题转换示例

原文：
```
3. 3 比 2 大。（  ）
答案：正确
```

转换结果：
```json
{
  "id": "q3",
  "type": "judgment",
  "question": "3 比 2 大",
  "options": ["正确", "错误"],
  "answer": true,
  "analysis": "",
  "points": 5
}
```

### 填空题转换示例

原文：
```
4. 4 + ____ = 7，____ - 2 = 5
答案：3，7
```

转换结果：
```json
{
  "id": "q4",
  "type": "fill",
  "question": "4 + ____ = 7，____ - 2 = 5",
  "answer": ["3", "7"],
  "analysis": "",
  "points": 10
}
```

### 简答题转换示例

原文：
```
5. 请写出5可以分成哪两个数？
参考答案：1和4，2和3
```

转换结果：
```json
{
  "id": "q5",
  "type": "essay",
  "question": "请写出5可以分成哪两个数",
  "answer": "1和4，2和3",
  "analysis": "",
  "points": 10
}
```

## 完整输出格式

最终输出必须是一个完整的 `questions.json` 文件内容：

```json
{
  "title": "从原文提取的题目集名称，没有则设为空字符串",
  "questions": [
    // 所有转换后的题目按顺序放入此数组
  ]
}
```

## 禁止行为

- 不要添加原文中没有的解析
- 不要修改原文中的题目措辞
- 不要自行推断答案（如果原文没有明确给出答案，answer 设为空值）
- 不要给简答题写"参考答案"，如果原文没有提供
- 不要给题目添加编号（如"1."），id 字段已经负责编号
- 不要在 options 中自行添加"以上全部"等选项，除非原文有
